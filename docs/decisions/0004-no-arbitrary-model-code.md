# 0004：不执行模型生成的任意代码

## 背景

让模型生成 JavaScript、TypeScript 或系统命令可以快速扩展能力，但也允许它绕过游戏输入、安全停止、权限和结果观察边界。

## 决定

MineIntent 不执行模型生成的任意程序或系统命令。模型只能调用实现方明确注册、带参数约束的接口；Runtime 负责解释调用并控制它能触及的游戏能力。

实验可以快速增加临时工具，但工具仍必须由项目代码实现。实验存在不表示该工具已经成为长期公共接口。

## 理由

- 可调用面明确，便于检查是否超过普通玩家能力。
- Runtime 可以在断线、取消或超时时停止身体输入。
- 动作结果可以回到观察与模型，而不是由任意脚本自行宣布成功。
- 模型幻觉不会直接获得操作系统或无限制 Mineflayer 权限。

## 风险

- 工具过窄会把同伴退化成固定命令集合。
- 工具过高会隐藏共同经历和身体过程。
- 每增加一种长期能力，都需要实现和真实环境验证。

## 复审条件

D40 用相对 look 和短时按键 move 验证：不执行任意代码时，模型是否仍能通过多轮反馈完成最小自然行为。
