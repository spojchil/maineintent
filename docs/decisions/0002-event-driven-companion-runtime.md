# 0002：采用事件驱动的持续同伴运行时

## 背景

Minecraft 世界持续变化。玩家可能在模型思考或同伴行动时聊天、改变想法或遇到危险；同伴也不能靠模型逐 tick 轮询维持存在。

## 决定

MineIntent 使用事件驱动的持续运行时。玩家消息、Minecraft 变化、动作结果、生命周期和危险可以触发处理；需要实时性的身体控制由本地代码执行。

该决定不预先规定完整事件目录、存储模型、动作 envelope 或并发框架。只有真实场景需要的机制才逐步加入。

## 理由

- 玩家可以在动作进行中介入。
- 模型不需要持续轮询世界。
- 动作后的真实结果可以触发下一次判断和语言。
- 连接、死亡和重启可以作为正常运行事件处理。

## 风险

- 事件过细会制造日志和实现负担。
- 事件过粗会丢失打断、失败和因果信息。
- “事件驱动”本身不能解决动作串行和玩家中断语义。

## 复审条件

D40 应直接观察模型工具往返、短时移动和玩家中途说话。根据实际失败再决定需要哪些队列、取消和状态记录。
