## 1. 阶段目标

v0.2 不扩展通关、建造或战斗能力。它要证明同伴的语言、感知、身体控制和结果验证共享同一套事实：同伴只能声称自己合理看见、亲自做过且经过验证的事情，普通规划不能利用角色尚未感知或探索的完整区块地图。

完成后，以下链路必须可信：

```text
玩家表达
→ 当前身体视角与已知空间消解目标
→ 注视和移动取得可交互准星
→ 发出协议动作
→ 区分客户端预测与服务端反馈
→ 验证方块、掉落物和背包结果
→ 基于证据自然回应并形成记忆
```

## 2. v0.1 联调事实

| 现象 | 已确认原因 | 影响 |
|---|---|---|
| “看向我”只回复，头部不转 | 没有 gaze skill；`attention` 不驱动身体 | 语言与身体分离 |
| 玩家在背后仍回答“看得见” | 模型只收到 `trackedPlayers`，没有自身朝向、FOV 或遮挡 | 视觉事实幻觉 |
| 六条 `block_dug` 中只有两块被服务端记账 | `bot.dig` 乐观更新本地世界，应用把 Promise 完成当作成功 | 动作证据失真 |
| 首次真实破坏两块但拾取为零 | 没有掉落物追踪与拾取闭环 | 破坏与采集混淆 |
| “面前/右边的树”可能切换到其他树 | skill 只有数量和半径，没有已感知目标引用 | 空间指代丢失 |
| 树叶后的原木被直接尝试 | 距离检查没有准星射线和可见方块面 | 交互不符合客户端过程 |
| 半透明方块行为不一致 | `transparentHint` 未消费；光学、交互、碰撞和导航共用形状 | 视觉与行动策略混淆 |
| 能直接找到隐藏树间路线或迷宫出口 | Pathfinder 读取完整 `bot.world` 执行 A* | 普通规划具有上帝视角 |
| 无法从日志复原路径和服务端回滚 | 缺少候选、射线、路径、预测、服务端反馈和拾取阶段事件 | 调试不可证伪 |
| DeepSeek 首轮 JSON 层级无效 | 提示只给协议名，没有合法结构示例或一次修复 | Provider 可靠性不足 |

第二次采木由服务端确认累计破坏从 2 增至 6、拾取从 0 增至 4，证明底层能力可以工作，但成功依赖场景，且当前日志无法可靠解释成功与失败差异。

## 3. 范围

### 本阶段负责

- Mineflayer 协议驱动、Safety Control、Perception、Epistemic Map 和 Motor Controller 的边界；
- 可复原的动作证据链和仅用于测试的 Paper 服务端裁判；
- 第一人称 FOV、实体可见性、光学材质与空间指代；
- 注视玩家、位置和方块面的可验证动作；
- 以准星、服务端方块变化、掉落物和背包为证据的采木闭环；
- 受认知地图约束的普通寻路与未知区域探索；
- 语言事实门控、部分成功和失败报告；
- 固定场景的单元、Paper 集成与真人观察验收。

### 非目标

- 完整像素渲染、截图视觉模型或 OCR；
- 完整通关、建造、成熟战斗和任意工具链；
- 为任务成功保留全知路径作为普通回退；
- 依赖生产服务器 OP、RCON、插件或文件访问；
- 在本阶段替换全部 Mineflayer 协议实现。

## 4. 实施顺序

### P0：事实与可观测性

1. 定义 `commanded → client_predicted → server_observed → outcome_verified` 证据状态。
2. 记录 skill phase、目标候选、射线、路径请求、重规划、协议动作和结果验证。
3. 建立只在本地和 CI 启用的 Paper 测试裁判，记录方块破坏和物品拾取的玩家、坐标与时间。
4. 在固定低树和叶片遮挡场景重现真假挖掘，先让失败可复原。

P0 完成前不增加新的复杂动作。

### P1：第一人称感知与注视

1. 实现 `PerceptionViewport`、FOV、实体包围盒采样和光学 DDA。
2. 将在线、协议追踪、当前可见、部分可见、刚失去视野和未知分开。
3. 将碰撞、视觉遮挡、交互遮挡和导航效果拆成版本化材质策略。
4. 实现 `look_at_player`、`look_at_position`、`look_at_block_face`，验证最终 yaw/pitch 和准星目标。
5. Context 只接收 Cognitive Observation，不再直接接收 `trackedPlayers` 精确表。

### P2：挖掘、拾取与目标树闭环

1. 将“面前/右边/这棵树”消解为带 observation evidence 的目标引用。
2. 候选原木必须有可见方块面；准星首个命中不是目标时不得发送挖掘。
3. 替换 `bot.dig` 的成功语义；本地乐观空气不产生领域成功事件。
4. 跟踪服务端方块变化、掉落物、移动拾取和背包增量。
5. 将树建模为当前可见和交互发现的结构，优先低处原木，有限处理叶片、换站位和失败目标。
6. 支持零成功、部分成功、目标切换与拾取失败的诚实终态。

### P3：认知地图与真人式普通寻路

1. 建立当前观察、探索历史、记忆和未知四种空间状态。
2. 普通 Intentional Planner 只使用 Epistemic Map；原始区块只参与硬碰撞与跌落安全。
3. 禁止未探索迷宫的全图路线；提供观察、试探岔路和回退行为。
4. 已走过路线可以形成带世界、维度、时间和置信度的空间记忆。
5. 为日常行为关闭无必要跑酷，限制绕路长度，并记录路径来源与已知性。

### P4：语言事实门控与阶段验收

1. “看见、到达、破坏、拾取、完成”分别绑定视觉、位置、服务端方块、背包和动作终态证据。
2. 动作开始前只允许协调语言；结果语言在验证后生成。
3. Model Provider 支持严格 schema 和至多一次结构修复，不把修复失败伪装成理解成功。
4. 运行全部自动测试和真人观察清单，关闭阻断缺陷后再扩展新技能。

## 6. 阶段完成定义

1. 玩家位于身后时，同伴回答“不在当前视野”；执行“看向我”后角色头部转动且视觉证据变为可见。
2. 玻璃、树叶、水、植物和实心方块在视觉、交互、碰撞与导航中有分别测试的策略。
3. 叶片遮挡的原木不产生假 `block_break_confirmed`；服务端拒绝会被明确记录。
4. “砍面前这棵树”不会静默切换到其他树；若必须换目标，先说明并等待适当协调。
5. 破坏、掉落和拾取分别验证；背包未增加时不能报告“收集到了”。
6. 首次进入固定迷宫时需要探索；完成探索后重新进入可使用已形成的路线记忆。
7. 任一动作可以从玩家消息追到模型决策、目标证据、身体阶段、协议反馈、服务端裁判和最终语言。
8. Paper CI 在全新模板世界中稳定运行，生产运行不依赖测试插件或管理权限。

## 7. 实施保护

- v0.1 的聊天、活动、记忆、动作资源锁、取消和 Backend 生命周期继续复用。
- 不以更强模型替代感知、动作验证或知识边界。
- 不通过增加寻路超时掩盖错误目标和未知地图。
- 不把 Paper 测试裁判输出注入生产同伴认知。
- ADR 0005 未接受前，除 P0 诊断外不扩展依赖 Mineflayer 高层自动化的新技能。
