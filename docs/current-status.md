# 当前项目状态

## 主线重置

本次工作从 `main@8ab08fd` 创建，同时收缩公开文档和仍在运行的实验代码：移除已经失去生产消费者的 Action/Skill/Pathfinder 链、本地 Paper 运维文件与旧模型协议，建立 D40 所需的短时身体工具闭环。

主线不再保留旧目标架构、旧路线图、未接受的信息接口全集和已经退役的实验说明。这不表示过去从未发生，也不表示旧文档中的每个判断都错误；它只表示这些材料不再冒充当前实现或当前排期。需要追溯时仍可从 Git 历史读取。

当前继续保留的长期基线只有：

- [产品设计](./product-design.md)；
- 第一版使用 Mineflayer；
- 事件驱动的持续同伴方向；
- 不执行模型生成的任意代码。

## 实现事实

MineIntent 仍是原型。本次只新增相对转头和短时按键移动；没有新增导航、挖掘、战斗、GUI 或长期自主能力，也没有证明当前代码适合长期生存陪伴。

当前模型和身体接口仍在实验中。旧的 Grounding、Behavior Synthesizer、Action Runtime、完整 Information/UI 设计和 Pathfinder 路线都不再被主线文档声明为最终结构；是否保留其中任何机制，要看后续真实实验。

D40 的模型终态只允许生成聊天回复。现有记忆存储和检索代码继续保留，但这次不开放模型写入，避免把尚未验证的动作叙述当成共同经历固化。

## D40 进展

当前实验把一个很小但完整的场景跑到底：

~~~text
“看看那只羊”
→ 读取当前视野
→ 模型调用相对 look
→ Runtime 返回动作后的新视野
→ 模型必要时继续调整
→ 使用真实按键语义 move
→ 观察位移、无效果、打断和最终回应
~~~

首轮观察已经暴露三个事实：

- 一次端到端往返约 16 秒；
- 该轮 token 规模约 40k，具体计量口径仍需固定；
- 模型在证据不足时声称“羊跑掉了”，说明反馈回路能提供纠正材料，但不会消除模型幻觉。

这些数字是实验观测，不是性能基线。当前尚未通过重复样本区分冷启动、模型推理、上下文体积、工具往返和游戏观察各自占用的时间与 token。

## 尚未决定

- 动作如何串行，以及玩家中途说话时怎样打断；
- 撞墙、没有位移和目标离开视野怎样反馈给模型；
- 当前体素与实体投影是否足以支持自然观察；
- 何时需要更高级的移动或导航工具；
- 哪些内部动作标识、世界 epoch 和目标锁定只由 Runtime 管理；
- 多轮模型往返的速度和费用是否达到可用体验。

下一步不是先写完整架构，而是重复运行 D40、记录失败，并由实际结果决定哪些机制值得进入主线。
