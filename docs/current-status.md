## D40 当前能力

```text
主要玩家发给同伴的聊天
→ 当前状态、背包、声音与第一人称视野
→ 模型调用 look_relative 或 move_input
→ Runtime 执行短时身体输入
→ 返回实际效果和动作后新视野
→ 模型继续判断或回复
```

- `look_relative` 的 yaw/pitch 单次限制在 ±90°。
- `move_input` 只能按住前、后、左、右之一 50–1500 ms，可选疾跑。
- 所有模型可见位置使用 `[右, 上, 前]` 相对坐标；不暴露世界坐标、实体 ID 或目标 ref。
- 工具结束后重新读取视野，并返回实际转向或位移效果。
- 最终模型输出只能是聊天回复。旧记忆仍可检索，本实验不新写长期记忆。

## 聊天、选择与硬失效

普通聊天按到达顺序进入模型。“停下”“等等”“别动”没有关键词分支，不会让 runtime 直接释放按键，也不会得到固定确认。模型可以照做、拒绝、延后，或在危险中先选择它认为更安全的动作；所有游戏内聊天回复都来自模型。

当前 `move_input` 是 50–1500 ms 的短原子输入，聊天队列会等待这一轮动作—观察闭环完成。持续十秒等长动作尚未实现；它们以后应作为可查询、可终止的后台 job 运行，让模型在处理下一条消息时看到 job 与当前世界状态后自行决定继续或终止，而不是让消息自动抢占。

只有断线、死亡/重生、换世界/维度、应用停止或请求超时等执行上下文已经失效的情况才会硬取消。硬取消会：

- 使尚未开始的旧世界聊天和当前模型 `runId` 失效；
- 释放所有移动键；
- 取消尚未发送的旧聊天分段；
- 通知 Python Agent Service，并中断本地上游模型 socket；
- 阻止迟到的工具结果或模型回复再产生身体动作或发言。

远端供应商在 TCP 断开后是否立即停止内部推理，仍取决于供应商实现。

## 尚未验证

- 没有实现高级移动、自动导航、跳跃、挖掘、战斗、GUI 或长期自主。
- 没有后台 action job；不要把 `move_input` 的时长扩成阻塞模型的长动作。
- 当前没有由受伤或怪物自动触发的新模型轮次；模型只会在处理聊天时从当前状态和视野判断风险。
- 尚未在此候选分支上重新运行真人 + Paper + 真实模型的 D40 场景。
- 仓库尚无可脱敏复盘的逐轮 D40 trace；因此不把之前单次观察的延迟、token 或语言错误当成性能基线。
- 现有 v0.2/v0.3 Issue 和 milestone 仍描述旧路线，在 D40 形成可重复证据以前不由本 PR 批量关闭或改写。

下一步是重复运行真实场景，记录模型实际可见信息、每次工具调用和耗时、动作后观察、最终聊天、token 与取消结果，再决定哪些机制值得长期保留。
