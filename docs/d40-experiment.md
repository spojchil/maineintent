# D40：look + move 端到端实验

## 目的

D40 不先决定新的模型—身体架构。它用一个可观察场景回答：模型能否在没有模型可见世界目标 ref 的情况下，通过动作、观察和再次决策表现出最小的自然身体闭环。

目标场景：

~~~text
玩家说“看看那只羊”
→ 模型获得当前视野
→ 调用相对 look
→ Runtime 返回新的视野
→ 没对准就再次调整
→ 通过真实按键语义 move
→ Runtime 返回位移、无效果或中断后的观察
→ 模型基于实际结果自然回应
~~~

## 两种移动能力

实验先提供低级 move：它表达普通玩家按住前、后、左、右之一的短时输入，由 Runtime 负责释放按键、取消和超时。它是身体控制能力，不是“走到羊旁边”之类的任务宏；当前没有跳跃输入。

未来可以同时向模型提供更高级的移动或导航工具，用于降低长距离移动的模型往返延迟和费用。高级工具不会因此替代低级输入；两者可以面向不同距离、精度和中断需求共存。高级工具的目标表示、规划方式和可信边界不由本实验预先确定。

## 实验边界

- 模型不管理实体 ID、方块 ref 或世界坐标目标锁。
- Runtime 可以内部保留 action ID、连接/世界 epoch、短时目标锁和控制状态。
- 当前实现允许直接、一次性接线，不先补齐幂等、resource lease 或完整结果协议。
- 工具返回实际观察，不判断玩家的高层意图已经达成。
- 模型幻觉不可避免；新的动作结果用于纠正下一轮判断。
- 本实验的最终模型输出只有聊天回复，不允许把未经验证的动作叙述直接写入长期记忆；记忆模块继续保留，但等出现值得记录且能绑定证据的共同经历后再接回。
- Paper 裁判信息只用于测试断言，不能进入模型上下文。

## 首轮观察

第一次端到端尝试已经暴露：

- 一轮往返约 16 秒；
- token 规模约 40k，但输入、推理与输出的计量口径尚未冻结；
- 模型错误声称“羊跑掉了”，当时观察证据不足以支持这句话。

这说明“动作后再观察”不会自动解决语言幻觉。后续运行必须同时保存模型可见视野、工具参数、工具结果和最终聊天，才能判断错误来自观察不足、上下文噪声还是模型自行补全。

单次数字不能证明延迟稳定，也不能证明慢就是可信行为。需要重复样本，并把冷启动、模型推理、工具往返、视口生成和游戏动作分别计时。

## 要观察的问题

- 相对方块与实体投影是否足以找到指定的羊；
- look 需要几轮才能对准，头部运动在其他玩家看来是否自然；
- 低级 move 的调用频率、延迟和 token 是否可接受；
- 连续工具应串行到什么程度；
- 玩家在移动中发消息时能否及时打断或转向；
- 撞墙、原地踏步、掉落和目标离开视野怎样如实反馈；
- 模型是否依据新观察纠正，还是继续编造解释；
- 何种情况下需要新增高级移动工具。

## 结果记录

每次运行至少记录：场景初始条件、模型实际可见信息、每次工具调用与耗时、玩家可见动作、动作后观察、最终聊天、token 用量、取消和清理结果。

D40 的成功不是“这次恰好走到羊旁边”，而是让上述问题产生可重复证据。实验完成后，再由结果决定保留、替换或删除哪些临时代码与接口。
